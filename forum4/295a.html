content 我觉得GoAgent的RC4加密未必靠谱，顶多起到混淆流量的作用，原因我觉得是因为多次使用同一个密钥。对于RC4这种序列加密算法（Stream Cipher），两次加密不能够使用同一个密钥，主要是因为同一个密钥，算法产生的伪随机密钥流是一样的。而RC4加密的方式，就是伪随机密钥流T与明文P进行异或（XOR），得到密文C，解密的时候，密文C跟密钥流T异或得到明文P。假设你使用同一个密钥加密两个明文P1，P2，得到C1，C2，就有C1 = P1 xor T，C2 = P2 xor T，将C1与C2异或，就有 C1 xor C2 = (P1 xor T) xor (P2 xor T) = (P1 xor P2) xor (T xor T) 【改变顺序不影响结果】， 注意根据异或的性质，T xor T = 0，T xor 0 = T，所以C1 xor C2 = P1 xor P2，假设攻击者破解了其中一个明文P1，那么明文P2也就知道了(P2 = C1 xor C2 xor P1)。
